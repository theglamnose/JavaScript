
# Notes on JavaScript (and TypeScript)
  
In the directory at hand you will find some notes, which I prepared while learning JavaScript.
Having several years of experience in the fields of functional and declarative programming I found
some concepts of JavaScript quiet irritating. Due to my observations I investigated some aspects
a little bit more and here are the results. Most of them are examples in JavaScript, but you will
also find some interesting TypeScript pieces of code.

## Contents

<dl>
  <dt><code>
  arrays_and_objects.js
  </code> </dt>
  <dd> 
Though these two data structures look very different at first glance, this text will show some
  surprising similarities (and explain the existence of "sparse arrays").
  (see also the file <code>sparse_arrays.txt</code>).
  </dd>
</dl>

<dl>
  <dt><code>
  call-by-value_or_-reference.js
  </code> </dt>
  <dd> 
  The basic question if you learn a new programming language. The text will give you the answer
  and will show you how to change the handling of function arguments.
  </dd>
</dl>

<dl>
  <dt><code>
  closure.ts
  </code> </dt>
  <dd> 
  Closure is one of the most powerful aspects implemented in JavaScript, which you definitely have to
  know.
  </dd>
</dl>

<dl>
  <dt><code>
  curry.js
  </code> </dt>
  <dd> 
  If you have worked as a functional programmer, the function transformations curry and uncurry are
  a handy tool for your work. JavaScript supports these transformations too.
  </dd>
</dl>
  

<dl>
  <dt><code>
  expressions.txt
  </code> </dt>
  <dd> 
  A short overview of the different kinds of expressions offered by JavaScript and also a short overview of the
  data types of JavaScript. 
  </dd>
</dl>

<dl>
  <dt><code>
  higher-order-modules.txt
  </code> </dt>
  <dd> 
  Though JavaScript was not designed to support higher-order-modules, it is possible (in some restricted way)  
  to simulate them in JavaScript. The text contains a small example of a higher-order-module (generic set of
  elements based on balanced trees) and gives a suggestion how to implement higher-order-modules.
  </dd>
</dl>
  
<dl>
  <dt><code>
  JSON_supported_constants.js
  </code> </dt>
  <dd> 
  JSON does not support all JavaScript constants. This small demo shows where JSON is not JS.
  </dd>
</dl>
  
<dl>
  <dt><code>
  recursion.ts
  </code> </dt>
  <dd> 
  This text explains the concepts of the problem solving technique called recursion by implementing the sorting
  algorithm named "quicksort". Also you will see a nice application of generic types in TypeScript.
  </dd>
</dl>

<dl>
  <dt><code>
  shallow_copy.js
  </code> </dt>
  <dd> 
  It seems, that JavaScript does not use pointers (or references), but in fact JavaScript uses them a lot.
  Shallow-copy is an effect based on references, which can lead to errors, which are very hard to find.   
  The text gives an example and shows how to get rid of it.
  </dd>
</dl>

<dl>
  <dt><code>
  sparse_arrays.txt
  </code> </dt>
  <dd> 
  Spares arrays are not so easy to handle. Here you will find a way to compute their length.
  (see also the file <code>arrays_and_objects.js</code>).
  </dd>
</dl>
