
   Expressions in JavaScript
   
An expression is a string, which can be interpreted as a value. This sounds quiet familiar,
because in our daily life we use often expressions to describe values. Here are some examples:

1. "forty two"
  This string describes the natural number 42 in English.
  
2. "neljäkymmentäkaksi"
  42 described by the finnish word.  
  
3. "0x2A"
  Describes the natural number 42 written with hexadecimal digits.  
  
4. "40 + 2"
  Describes a short computation, which yields the number 42. 
  
5. "The smallest natural number, that can be described exactly of two ways as a sum of two cubics." 
  Describes the natural number 1729 = 1^3 + 12^3 = 9^3 + 10^3 (it's called the Hardy–Ramanujan-Number).  
  
All these strings can be interpreted as legal descriptions, but in a programming language we only
accept strings, which can be interpreted by a computer with some kind of computation.
Due to this, we only accept strings as given in the items 3 and 4. To distinguish between a string
written inside of computer program and a constant of datatype string, from now on we will use quotes
only for string constants.

To define exactly the set of allowed expressions, we have to know, which kind of values JavaScript supports
and how these values are written as strings. This does not only influence the writing of constant values, but
also of structured constants.

An expression in JavaScript can be quiet complex, so we start with expressions, which play the role of atoms
(greek for "not divisible"). These atoms are called "primary expressions" and can be one of the following:
* a constant (e.g. 3.14, "Hello world!", null, true)
* a name of a defined variable

Given some primary expressions we can build some more complex expressions by using operators. The
predefined expressions can be grouped:
* "arithmetic expressions" (e.g. 2 + 40, 10**3, a % b)
* "comparison expressions" (e.g. 3 > 4, "abc" == "ABC")
* "logical expressions" (e.g. (0 < x) && (x < 10))

Instead of using a predefined operator you can define your own function and use its name in an
"invocation expression", which consists of the function name and a list of expressions, which deliver
the arguments to the function.
  E.g.:
	function add_one(n){ return( n + 1)}; // function declaration
	add_one(3); // invocation expression yields 4
	
With JS 2020 a flexible way for invoking optional arguments was defined:
* "conditional invocation": checks if an optional functional argument is given and invokes it if present
  (e.g. opt_fun?.(x,y))

With comparison expressions we are able to compare numbers, but there are more possibilities to compare values:
"relational expressions". Here is an overview of comparisons:
* equality and inequality (e.g. "1" == 1)
* property existence in an object (e.g. "name" in { name: "Joe", age:32})
* instance of a class (e.g. point instanceof GraphicObjects)

Usually you apply an assignment to change the value of a variable, but the interesting thing is, that the
assignment can be used as an expression: its value is the value of the right-hand side.
The following statement consists of three nested *assignment expressions*:
  a = b = c = 0; // all three variables set to 0.

JavaScript offers several ways to build up complex structures based on given ones:
* arrays (numbered sequence of items)
* objects (association list of key-value pairs)

For each of these structures expressions are defined. For arrays the following expressions can be used:
* "array initializer": creates a new array based on a given sequence of expressions
  (e.g. [], [1,2,3], [[],[x,y]])
* "access expression": yields an element of an array
  (e.g. a[0], matrix[row][column])
  
For objects the following expressions can be used:
* "object initializer": creates a new object based on a given key-value list
  (e.g. {}, {name:"Tom",surname:"Jones"})  
* "access expression": yields the value of a property (value of a key)
  (e.g. person.name, record["year of birth"])
  
JavaScript supports the object-oriented programming paradigm. Due to this we can use classes to construct
new objects based on a given class declaration. The construction of a new object can be done by an
* "object creation expression" (e.g. new Point(x,y))
The application of a method, which was defined in a class is realized by an
* "invocation expression" (e.g. Math.random())
  
Because a JavaScript program is executed by an interpreter, it is possible to construct JavaScript source code during run-time
and execute it. This is achieved by the *evaluation expression*, but due to security reasons browsers usually prohibit this.
  eval( "2 ** 3"); // result: 8.
